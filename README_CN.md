# 「怎样做成大事」项目决策框架

> **在花第一分钱之前，用数据而非直觉判断"该不该做"**

基于傅以斌《How Big Things Get Done》16,000+项目实证数据的项目决策框架。帮助你在启动任何项目之前，做出数据驱动的决策。

---

## 一分钟演示

```
你：我想做一个AI写小说的App，预算大概50万，团队3个人有2年开发经验

我：【判断】有条件Go — AI写作工具赛道已有成熟玩家，但垂直场景仍有空间

    【风险】
    1. 同质化风险：ChatGPT/Notion AI等已覆盖通用写作场景
    2. PMF风险：用户是否愿意为"AI写小说"单独付费？
    3. 版权风险：AI生成内容的版权归属尚未明确

    【下一步】用一周时间验证：在番茄小说/晋江社区找到10个愿意内测的用户

    💡 说"展开"看完整分析
    💡 说"工具"用风险评估计算器
```

## v5.0 核心改进

| 改进 | 解决的问题 |
|------|-----------|
| **追问机制** | 信息不足时先追问，不在模糊信息上做判断 |
| **模型驱动分析** | 5个心智模型从"知识"变成分析引擎，每次展开都逐项调用 |
| **可复现计算** | 风险评估公式公开透明，同样输入永远同样结果 |
| **方案对比** | 支持"A还是B"的多方案对比分析 |
| **用户画像自适应** | 自动识别创业者/企业/学生/技术团队，调整输出风格 |
| **案例动态匹配** | 展开分析时自动匹配最相关的成功/失败案例 |

## 核心设计

### 四层渐进式工作流

```
第一层：听懂你要什么 + 识别你是谁
    ↓
第二层：信息充足？→ 给判断 / 信息不足？→ 追问关键信息
    ↓ 说"展开"
第三层：搜索 → 模型分析(5个模型) → 案例匹配 → 综合报告
    ↓ 说"对比"
第三层变体：多方案对比（A vs B）
    ↓ 说"工具"
第四层：决策工具（风险评估计算器 / RCF速查 / 承诺谬误自检）
```

### 5个心智模型 = 分析引擎

| 模型 | 回答的问题 | 在分析中的角色 |
|------|-----------|-------------|
| 外部视角锚定 | 预期 vs 现实差距多大？ | 用搜索数据校准用户预期 |
| 承诺谬误 | 是否存在认知偏差？ | 5项自检清单 |
| 肥尾风险 | 最坏情况有多坏？ | 评估毁灭之窗暴露 |
| 模块化 | 能否拆解降低风险？ | 判断模块化可能性 |
| 慢思考快行动 | 节奏是否正确？ | 检查规划充分性 |

### 风险评估计算器（可复现公式）

```
得分 = 类型基准(40-65) + 预算因子(0-15) + 工期因子(0-15) - 经验(0-15) - 技术(0-15) - 参考(0-10)
≥70 🔴 高风险 / 40-69 🟡 中风险 / <40 🟢 低风险
```

## 搜索能力

集成 [AnySearch](https://github.com/anysearch-ai/anysearch-skill) 实时搜索引擎：

| 场景 | 并行查询 | 内容 |
|------|---------|------|

| 风险诊断 | 3个 | 项目风险、行业政策、止损案例 |

## 适用场景

**最适合**：大型基础设施、资本密集型不可逆项目、有历史先例的领域

**部分适用**：软件产品（+敏捷）、硬件产品（+精益创业）、创业项目（+PMF验证）

**不适用**：纯创意/艺术项目、个人小型任务、全新无先例的技术领域
