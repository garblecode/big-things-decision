# 「怎样做成大事」框架提炼文档

## 一、心智模型提取（5个核心模型）

基于三重验证筛选（跨域复现、生成力、排他性）：

### 模型1: 慢思考快行动（Think Slow, Act Fast）
- **一句话描述**: 在风险成本低廉的规划阶段充分迭代验证，在资本投入后的执行阶段快速推进以关闭"毁灭之窗"
- **跨域复现**: 帝国大厦（纸上设计数月→10天14.5层）、皮克斯（2700张故事板→快速制作）、亚马逊（先写新闻稿→再开发）
- **生成力**: 可推断：任何不可逆项目都应先进行低成本实验，再进入高成本执行
- **排他性**: 与传统"边做边改"和纯敏捷方法形成对比
- **来源证据**: 傅以斌16,000+项目统计；卡尼曼《思考，快与慢》理论转化
- **应用方式**: 规划阶段用纸上作业/数字孪生/原型迭代消除不确定性；执行阶段压缩时间表
- **局限性**: 不适用于可逆、可快速迭代的软件项目；过度规划可能错失市场时机

### 模型2: 外部视角锚定（Outside View Anchoring）
- **一句话描述**: 用历史参考类项目的分布数据作为预测锚点，而非从零开始内部加总估算
- **跨域复现**: 厨房改造参考邻居花费、RCF在英/丹/香港交通项目中的应用、傅以斌拯救香港高铁项目
- **生成力**: 可推断：任何预算预测都应先找同类项目的历史分布，再调整
- **排他性**: 直接对抗"规划谬误"和"独特性陷阱"——项目发起者总认为自己的项目独一无二
- **来源证据**: 卡尼曼称RCF为"提高预测准确性最重要的建议"；英国政府强制要求使用
- **应用方式**: 6步操作法（识别参考类→收集基准率→以分布为锚→谨慎调整→设定置信水平→透明记录）
- **局限性**: 真正新颖的项目（前所未有的技术）缺乏参考类；可能产生"自我实现预言"式预算膨胀

### 模型3: 模块化分解（Modular Decomposition）
- **一句话描述**: 将大型项目拆分为可重复、可测试、可学习的小型标准化单元
- **跨域复现**: 尼泊尔学校（乐高单元）、帝国大厦（预制构件）、特斯拉Gigafactory（21个小工厂）、丹麦风电（标准化涡轮）
- **生成力**: 可推断：任何大项目都应寻找"乐高块"——可重复的子单元
- **排他性**: 与"一次性巨型项目"（如传统核电站、悉尼歌剧院）形成鲜明对比
- **来源证据**: 集装箱革命（运输效率提升80%+）；日本文殊核电站反例（150亿$零发电）
- **应用方式**: 设计可重复模块→工厂化预制→现场组装→学习曲线效应
- **局限性**: 前沿科学项目（韦伯望远镜）本质需定制；过度模块化可能牺牲创新性

### 模型4: 肥尾风险认知（Fat Tail Risk Awareness）
- **一句话描述**: 复杂项目的结果呈肥尾分布——极端事件（灾难性超支）的概率远高于正态分布预测
- **跨域复现**: 奥运会项目、大型基建（平均超支62%）、COVID-19对项目的影响、气候变化
- **生成力**: 可推断：必须为极端情境做准备，而非仅考虑"平均情况"
- **排他性**: 对抗传统项目管理中的"正态分布假设"和"均值回归"思维
- **来源证据**: 傅以斌2020年"回归尾部定律"论文；苏格兰议会大楼超支978%
- **应用方式**: 识别毁灭之窗→压缩执行期→设计冗余→预聘专家（如英国高铁预聘全国考古学家）
- **局限性**: 过度防范可能导致"分析瘫痪"；难以量化极端事件的具体概率

### 模型5: 承诺谬误识别（Commitment Fallacy Recognition）
- **一句话描述**: 区分"欺骗"（策略性虚假陈述）与"妄想"（乐观偏差），识别过早承诺导致的锁定效应
- **跨域复现**: 悉尼歌剧院（政治驱动开工）、加州高铁（"首付款"心态）、五角大楼（3天草图）、电影产业（沉没成本）
- **生成力**: 可推断：项目启动前应检查是否存在"不得不做"的政治压力或沉没成本陷阱
- **排他性**: 傅以斌2005年核心学术贡献——首次系统区分"欺骗vs妄想"两个根源
- **来源证据**: 前旧金山市长Willie Brown名言；卡尼曼"规划谬误"研究
- **应用方式**: 检查清单（是否存在政治压力？是否低报预算？是否有替代方案？）→应对欺骗需问责机制，应对妄想需RCF
- **局限性**: 政治现实往往使"充分规划"难以实现；制度性激励可能压倒个人理性

## 二、决策启发式提取（8条核心规则）

可表述为"如果X，则Y"，有具体案例支撑：

1. **如果项目预算基于内部加总估算，则参考同类项目历史分布进行锚定**（案例：波士顿隧道超支300% vs 外部视角法的准确性）

2. **如果项目需要定制设计，则先验证可建造性（数字孪生/原型测试），再动工**（案例：毕尔巴鄂博物馆CATIA验证 vs 悉尼歌剧院边建边拆）

3. **如果执行期超过12个月，则识别并预聘关键领域专家防范黑天鹅**（案例：英国高铁预聘全国考古学家）

4. **如果项目声称"独一无二"，则强制寻找参考类——任何项目都是"其中之一"**（案例：傅以斌2025年HBR"独特性陷阱"论述）

5. **如果团队缺乏经验，则拒绝承接或引入经验丰富的"建筑大师"**（案例：乌松38岁首作失败 vs 盖里62岁成功）

6. **如果存在政治压力要求"尽快动工"，则启动承诺谬误检查清单**（案例：悉尼歌剧院"先挖洞"策略、加州高铁逐步缩水）

7. **如果项目可分解为重复性子单元，则采用模块化设计**（案例：尼泊尔学校提前8年完工、火神山10天建成）

8. **如果直觉与历史数据冲突，则站在历史一边——历史几乎总是对的**（案例：RCF预测准确度比传统方法高30+百分点）

## 三、表达DNA分析

作为主题Skill（非人物Skill），表达DNA应体现：

| 维度 | 特征 |
|------|------|
| **句式偏好** | 数据驱动陈述（"99.5%的项目..."）、对比结构（"A vs B"）、条件判断（"如果X，则Y"） |
| **词汇特征** | "肥尾"、"毁灭之窗"、"乐高"、"锚定"、"锁定"、"承诺谬误"、"外部视角" |
| **节奏感** | 先摆数据（震撼性统计）→再讲案例（具体故事）→再给原则（可操作建议） |
| **确定性表达** | 高确定性（"铁律"、"几乎总是"），但明确标注局限性（"不适用于..."） |
| **引用习惯** | 频繁引用卡尼曼、塔勒布、具体项目数据、历史案例 |
| **风格定位** | 学术严谨性 + 实践可操作性；数据驱动 + 故事化表达 |

## 四、价值观与反模式

### 核心价值观（排序）
1. **诚实面对不确定性** > 虚假精确
2. **历史数据** > 直觉判断
3. **可复制性** > 独特性宣称
4. **经验** > 创新冒险（在大型项目中）
5. **预防** > 应急（规划阶段）

### 反模式（明确反对）
- 反对"边做边改"的侥幸心态
- 反对"这次不一样"的独特性陷阱
- 反对政治驱动的过早承诺
- 反对"首付款"式策略性低报预算
- 反对忽视肥尾风险的正态分布思维

### 内在张力（方法论本身的矛盾）
1. **慢思考 vs 市场时机**: 充分规划可能错失窗口期
2. **模块化 vs 创新性**: 标准化可能抑制突破性创新
3. **历史依赖 vs 技术变革**: 快速变化领域历史参考类可能失效
4. **经验至上 vs 新人机会**: 过度依赖经验可能排除年轻创新者

## 五、诚实边界

1. **不能预测全新技术领域的项目结果**（如AI、量子计算等前沿项目缺乏历史参考类）
2. **不能替代项目发起者的政治判断**（方法论无法消除制度性激励导致的策略性虚假陈述）
3. **不能保证创意项目的成功**（方法论更适合可预测、可重复的工程类项目）
4. **不能消除所有黑天鹅风险**（只能降低概率，无法完全预防）
5. **信息截止到2023年**（原书出版年份），2020年后的模块化新案例（如SMR）需持续更新

## 六、智识谱系

**直接影响傅以斌的思想来源**:
- Daniel Kahneman & Amos Tversky → 规划谬误、系统1/2、乐观偏差
- Nassim Nicholas Taleb → 肥尾分布、黑天鹅、反脆弱
- Aristotle → Phronesis（实践智慧）
- Michael Polanyi → 隐性知识
- Michel Foucault → 权力与理性

**傅以斌影响的方向**:
- 英国/丹麦/香港政府 → RCF强制政策
- 项目管理实践界 → 大型项目风险评估标准
- 学术界 → 实践性社会科学方法论、回归尾部定律

**在思想地图上的位置**:
- 介于行为经济学（认知偏差）与项目管理（实践工具）之间
- 介于学术理论（Phronetic Social Science）与政策咨询（RCF）之间
- 介于悲观主义（肥尾风险）与建设性行动（11条启发式）之间
